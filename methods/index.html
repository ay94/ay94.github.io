Multilayered Topic Modelling

Hierarchical narrative discovery at scale

Placeholder — describe this method in detail here. Explain the conceptual approach, the specific techniques used, and why this method is well-suited for the kinds of problems you apply it to.

Multilayered topic modelling uses a hierarchical approach to discover themes at multiple levels of granularity within a corpus. At the top level, broad thematic clusters are identified. Each cluster is then modelled independently to surface more granular sub-topics, enabling a structured view of discourse from macro-narratives down to specific talking points.

This approach is particularly valuable for large, heterogeneous corpora where both broad thematic mapping and fine-grained narrative analysis are required — for example, in disinformation research where understanding both the overarching campaign framing and the specific tactical sub-narratives is essential.

NER Evaluation Workflow

Three-stage benchmarking across low and high resource languages

Placeholder — describe this method in detail here. Explain the three stages of the workflow and what each stage produces.

The NER evaluation workflow comprises three stages: baseline measurement, error taxonomy classification, and comparative cross-language analysis. In stage one, standard precision, recall, and F1 metrics are computed across entity types and datasets. In stage two, errors are classified into a structured taxonomy — distinguishing boundary errors, type confusion, spurious entities, and missed entities.

Stage three positions a model's performance within a comparative landscape by benchmarking against reference models on the same data, with results stratified by language resource level (high-resource vs. low-resource), script type, and entity category. This three-stage structure moves evaluation from a single aggregate number to an actionable diagnostic profile.

Machine Translation Evaluation

Comparative metric framework with BERTScore as primary signal

Placeholder — describe this method in detail here. Why is BERTScore used as the primary metric, and how does it complement other evaluation signals?

The machine translation evaluation framework uses BERTScore as the primary automated metric, supplemented by BLEU and chrF as secondary signals. BERTScore is preferred because it captures semantic similarity through contextualised embeddings rather than surface-level n-gram overlap, making it more robust for morphologically rich languages and domain-specific text.

The framework also incorporates structured human evaluation on a sampled subset, using a rubric that assesses adequacy and fluency independently. Results from automated and human evaluation are compared to identify systematic discrepancies and calibrate the automated metrics against human judgement.

Sentence-level Sentiment Analysis

Fine-grained investment sentiment detection and driver classification

Placeholder — describe this method in detail here. How does sentence-level granularity differ from document-level sentiment, and why does it matter for the use cases you apply it to?

Sentence-level sentiment analysis operates at the level of individual sentences rather than entire documents, allowing sentiment to be attributed to specific claims, entities, or events within a longer text. This granularity is essential for financial and geopolitical text, where a single document may express positive sentiment toward one actor and negative sentiment toward another.

The workflow involves sentence segmentation, sentiment classification (positive / negative / neutral), and driver classification — identifying the specific topic or entity driving the sentiment in each sentence. Results are aggregated to produce entity-level sentiment profiles, time-series sentiment trends, and comparison across sources or regions.
