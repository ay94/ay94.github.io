BBC Monitoring

Broadcast Transcript Analysis & NER Pipeline

Completed · 2023

The Project

Context & Objectives

BBC Monitoring tracks and analyses broadcast media globally — including Arabic language television and radio. CASM Technology was commissioned to build a pipeline that could process AWS Transcribe outputs (automatic speech recognition transcripts) and extract structured intelligence from them — specifically named entities, speaker attribution, and machine translation for non-English content.

My Role

Contributions

Technical Lead and sole builder. I designed and built the full transcript analysis pipeline from scratch, packaged it as an installable Python library (transcript_analysis v0.2.1), and built a Streamlit dashboard for analysts to explore and query the outputs.

Method

Approach & Pipeline

  1. Transcript parsing — AWS Transcribe JSON outputs parsed and structured
  2. Speaker diarisation — speaker segments identified and attributed (spk_0, spk_1 etc.)
  3. NER extraction — NamedEntityExtractions class built with HuggingFace model loading (supports any model including Arabic CamelBERT), batch processing via PyTorch DataLoader, subword token merging, and JSON schema output (PER, LOC, ORG, MISC)
  4. Machine translation — translated text integrated alongside original for bilingual display
  5. Streamlit dashboard — 4-tab interface: Transcript Viewer, Speaker Statistics, Speaker Data with highlighted NER, NER Analysis with filtering and top-entity charts
  6. Packaging — delivered as transcript_analysis v0.2.1, installable Python library

Outcomes

Results & Impact

Production-grade NER extraction pipeline built and deployed. Packaged as reusable Python library (transcript_analysis v0.2.1).

Streamlit dashboard enabling analysts to explore transcripts, speakers and entities interactively. Full Arabic NER support via CamelBERT integration.
