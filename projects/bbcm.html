BBC Monitoring

Broadcast Transcript Analysis & NER Pipeline

The Project

Context & Objectives

Placeholder — add project background here. Describe the commissioning organisation, the problem context, and what the project set out to achieve.

BBC Monitoring required a scalable pipeline to extract structured intelligence from large volumes of broadcast transcripts. The project aimed to automate entity recognition and enable rapid cross-transcript analysis.

My Role

Contributions

Placeholder — describe your specific responsibilities, what you built or designed, and how you contributed to the project outcomes.

Designed and built the end-to-end NER extraction pipeline. Handled transcript pre-processing, model fine-tuning on broadcast domain text, and structured output generation for downstream analysis.

Method

Approach & Pipeline

Placeholder — describe the technical approach: data collection, preprocessing, modelling choices, evaluation strategy, and tooling.

Fine-tuned a spaCy NER model on broadcast transcript data. Custom entity types were defined for people, organisations, locations, and broadcast-specific terminology. Output was structured as annotated JSON for integration with BBC Monitoring's existing systems.

Outcomes

Results & Impact

Delivered a production-ready NER pipeline integrated into BBC Monitoring's transcript processing workflow. Significantly reduced analyst time spent on manual entity extraction from broadcast content.
