BBC Monitoring

China's Public Diplomacy & Broadcast Intelligence

Completed · 2020—2023

The Project

Context & Objectives

A multi-phase engagement with BBC Monitoring spanning keyword-based discovery, full ML classification and broadcast transcript analysis. Phase 1 and 2 studied China's diplomatic social media activity across Twitter and Facebook covering 400,000+ messages from 370+ diplomatic accounts. Phase 3 built a production-grade transcript analysis pipeline for multilingual broadcast content including Indonesian, Iraqi Arabic and Farsi.

My Role

Contributions

Technical Lead across all three phases. Phase 1—2: built data collection infrastructure, engineered account master file parsing 13 handle types across 370+ diplomatic accounts, designed keyword-to-theme pipeline across 5 themes, trained 34 ML classifiers across English, Arabic, French and Spanish (average F1: 80.5%), coordinated 4 language annotation teams. Named contributor on the published BBC Monitoring / CASM Technology report. Phase 3: designed and built full transcript ingestion, preprocessing, speaker diarisation and NER extraction pipeline — packaged as installable Python library with Streamlit dashboard for quality validation.

Method

Approach & Pipeline

Phase 1 — Diplomatic Discovery & Keyword Analysis:

  1. Account master file — parsed 370+ diplomatic accounts across 13 handle types (embassy, ambassador, consulate accounts etc.)
  2. Data collection — Twitter and Facebook data collection via Twitter Stream API, CrowdTangle and Method52
  3. Keyword-to-theme pipeline — 5 themes designed and applied as keyword annotators across 400,000+ posts

Phase 2 — ML Classification (400,000+ posts, 4 languages):

  1. Annotation — coordinated 4 language annotation teams (English, Arabic, French, Spanish)
  2. Classifier training — 34 binary ML classifiers trained across languages and themes (average F1: 80.5%)
  3. Reporting — classified posts aggregated to produce China public diplomacy landscape analysis

Phase 3 — Broadcast Transcript Pipeline:

  1. Transcript parsing — AWS Transcribe JSON outputs parsed and structured
  2. Speaker diarisation — speaker segments identified and attributed (spk_0, spk_1 etc.)
  3. NER extraction — NamedEntityExtractions class built with HuggingFace model loading (supports any model including Arabic CamelBERT), batch processing via PyTorch DataLoader, subword token merging, and JSON schema output (PER, LOC, ORG, MISC)
  4. Machine translation — translated text integrated alongside original for bilingual display
  5. Streamlit dashboard — 4-tab interface: Transcript Viewer, Speaker Statistics, Speaker Data with highlighted NER, NER Analysis with filtering and top-entity charts
  6. Packaging — delivered as transcript_analysis v0.2.1, installable Python library

Outcomes

Results & Impact

Published BBC Monitoring / CASM Technology report on China's public diplomacy. 34 classifiers trained across 4 languages (average F1: 80.5%). Production-grade transcript pipeline deployed as installable Python library (transcript_analysis v0.2.1) with multilingual support.
